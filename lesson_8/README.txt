8 OOP. Dunder methods

Задание реализовал в двух модулях:
1. price_module.py -- модуль класса Price. Класс Price, создает объекты
класса Price, содержит переопределённые dunder methods -- __add__, __sub__,
__mul__, __truediv__ и __str__. Актуальный курс валюты берётся с помощью
HTTP-запроса к API на сайте https://www.alphavantage.co . При тестировании
обнаружил, что стороннее API даёт сбой на восьмом HTTP-запросе,
если запросы идут непрерывно. Для решения этой проблемы реализовал стратегию
которая не превышает количество запросов за один раз (модуль main.py не
предоставляет пользователю возможность перегрузить сторонее API HTTP-запросами).
Отдельный класс для реализации HTTP-запросов к API решил не создавать, так как
обошёлся четырьмя методами внутри класса Price.
2. main.py -- модуль предоставляет пользователю возможность интерактивно
создавать объекты класса Price и проводить с ними арифметические вычисления,
с учетом разных валют.
