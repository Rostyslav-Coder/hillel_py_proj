8 OOP. Dunder methods

Задание реализовал в двух модулях:
1. price_module.py -- модуль класса Price. Класс Price, создает объекты
класса Price, содержит переопределённые dunder methods -- __add__, __sub__,
__mul__, __truediv__ и __str__. Актуальный курс валюты берётся 
из словарика "EXCHANGE_RATE".
Пытался подтянуть курс с API, но т.к. не имеет значения откуда берется курс,
то будет из словарика "EXCHANGE_RATE".
2. main.py -- модуль предоставляет пользователю возможность интерактивно
создавать объекты класса Price и проводить с ними арифметические вычисления,
с учетом разных валют. В файле main.py использовал коментарии # type: ignore,
без них "mypy" сильно ругается.
