При планировании кода была выбрана стратегия, что ПО будет отрабатывать свой 
сценарий быстрее, если ПО будет:
1. Имитировать подключение к социальной сети;
2. Имитировать авторизацию на канале;
3. Публиковать сообщения с заданными задержками по времени;
4. Отключаться от условной социальной сети.
После этого все те же действия будут повторятся с другими условными социальными сетями.
Обязательным условием является то, что задержка по времени не должна превышать несколько секунд.
